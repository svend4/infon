## 🚀 Суть проекта TVCP

### Концепция
**Terminal Video Communication Platform** — видеосвязь через терминал с передачей данных в 10-20 раз меньше, чем Zoom/WebRTC.

### Технологический стек (планируемый)

#### 1. Блочный видеокодек .babe
- **Bi-Level Adaptive Block Encoding**
- Кодирует изображение в Unicode блочные символы (▀▄▌▐▖▗▘▝)
- **36 бит/блок**: 4 бита паттерн + 2×16 бит цвета (RGB565)
- **Битрейт:** 160-800 kbps (в 9× меньше Zoom)

#### 2. Yggdrasil P2P Mesh-сеть
- Децентрализованная IPv6 overlay-сеть
- End-to-end шифрование
- Без серверов, аккаунтов, DNS
- Самовосстанавливающаяся маршрутизация

#### 3. Терминальный рендеринг
- Unicode квадрантные блоки + ANSI True Color
- Разрешение: 80×60 "пикселей" в стандартном терминале
- Работает в iTerm2, WezTerm, kitty, Windows Terminal, GNOME Terminal

### Основные преимущества

| Параметр | Zoom | TVCP |
|----------|------|------|
| Минимальный канал | 1.8 Mbps | **200 kbps** (9× меньше) |
| Трафик/час | 540 MB - 1.62 GB | **~90 MB** (6-18× меньше) |
| GUI | Обязателен | **Терминал** |
| Центральный сервер | Да | **Нет (P2P)** |
| Аккаунт | Email | **Криптоключ** |
| RAM | 8-16 GB | **256 MB** |
| Работает headless | Нет | **Да** |
| IoT/Embedded | Нет | **Да** |

## 🎯 Целевые рынки

### Сегмент A: DevOps / SRE
- **Размер:** ~3.5 млн инженеров
- **Боль:** Управление headless-серверами, нужно переключаться из терминала в Zoom
- **Решение:** Видеозвонок прямо из SSH-сессии

### Сегмент B: IoT / Embedded
- **Размер:** ~200M устройств с камерой и Linux
- **Боль:** IP-камеры требуют 2-5 Mbps
- **Решение:** Видеопоток при 200 kbps

### Сегмент C: Спутниковая связь
- **Размер:** $300M рынок
- **Боль:** Спутниковый трафик $1-15/МБ. Час Zoom = $2700-8100
- **Решение:** Видеозвонок за ~$90-450 вместо тысяч

### Сегмент D: Приватность
- **Размер:** ~40M пользователей Signal
- **Боль:** Signal требует телефон и серверы
- **Решение:** Видеосвязь без аккаунта, серверов, метаданных

### Сегмент E: Military / Government
- **Размер:** $35B рынок
- **Боль:** Тактическая связь без зависимости от инфраструктуры
- **Решение:** Автономная mesh-сеть + E2E шифрование

### Алгоритм кодирования .babe

1. Разбить кадр на блоки 2×2 пикселя
2. k-means кластеризация (k=2) для каждого блока
3. Выбор из 16 глифов с минимальной ошибкой
4. Кодирование: [glyph 4 bit] + [fg RGB565 16 bit] + [bg RGB565 16 bit] = **36 bit/блок**
5. Delta-кодирование между кадрами
6. RLE-сжатие + Huffman

**Битрейт (80×60 @ 25 FPS):**
- I-frame: 5.4 КБ
- С delta: 2-4 КБ/кадр = 50-100 КБ/с = 400-800 kbps
- С RLE: ~20 КБ/с = **160 kbps** ✓
